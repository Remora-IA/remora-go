# WHY - Framework Echo

Echo existe porque automatizar sin entender el mundo real del usuario produce
software que nadie usa.

Echo guía una conversación de descubrimiento estructurada para construir un
árbol validado: AXIOM → THEORY → TASK → PAIN → OPPORTUNITY. Cada nodo viene
de una respuesta real, no de una suposición.

## Problema Que Resuelve

Sin Echo, el descubrimiento de procesos depende de la memoria del
entrevistador, de notas sueltas o de reuniones sin estructura. Echo persiste
cada hallazgo como nodo verificable con evidencia.

## Relación Con Otros Frameworks

- **Alfa** compila el árbol de Echo en un flujo ideal verificable.
- **Foco** puede disparar un ciclo de descubrimiento cuando hay ambigüedad.
- **Quine** clasifica a Echo como tipo inquisitivo + nodos-árbol.

Echo no compila specs. Alfa no descubre dolores. Cada uno hace su parte.
