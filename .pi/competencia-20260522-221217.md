## 1. Competidores

### PJUD.cl (pjud.cl) — competidor estructural (gratuito)

**Propuesta de valor (cita de homepage):**
> "Consulta unificada de causas" — portal oficial del Poder Judicial de Chile

**Pricing** _(consultado 2026-05-22)_:
- Tier bajo: Gratuito — sin costo
- Tier alto: Gratuito — sin costo

**A quién van dirigidos:** Cualquier ciudadano, litigante o profesional legal que necesite consultar causas del sistema judicial chileno.

**Fortaleza principal:** Es la fuente primaria de datos. Todo competidor depende de él. Acceso oficial, gratuito y sin restricciones de funcionalidad básica para usuarios individuales.

**Debilidad principal:** Sin acceso programático (no hay API pública documentada). Bloquea IPs de estudios jurídicos que superan 2,500 conexiones diarias — señal documentada por el Colegio de Abogados desde 2016. Interfaz diseñada para consultas individuales, no para gestión de carteras masivas. Requiere ClaveÚnica para causas reservadas. Sin alertas automáticas, sin agregación de vistas, sin exportación estructurada de datos.

## 3. Gaps que nadie cubre

- **API con SLA enterprise al PJUD:** Boostr.cl tiene API pero sin SLA, sin garantía de datos, sin soporte enterprise, sin Auth0, pagando por consulta. CaseTracking tiene integración directa con PJUD pero no la expone como API consumible por clientes. Ningún competidor ofrece acceso programático documentado, con autenticación, SLA, y precios predecibles. Este es el gap más significativo y structuralmente difícil de llenar.

- **Búsqueda por persona jurídica cruzada multi-jurisdicción:** Ningún competidor confirma explícitamente en su homepage la capacidad de buscar todas las causas activas e históricas de una persona jurídica cruzando simultáneamente todas las jurisdicciones (Civil + Penal + Laboral + Cobranza + Apelaciones + Suprema). Es funcionalidad crítica para due diligence legal.

- **Precios públicos en el segmento de volumen medio:** CaseTracking, relCase y LegalFlow no publican precios. Solo Pocket Lawyer (hasta 200 causas) y Boostr.cl (pay-per-use técnico) tienen precios visibles. Hay opacidad total en el segmento de estudios jurídicos medianos (200–2,000 causas), lo que genera fricción en el proceso de compra.

- **Acceso en tiempo real con Auth0 para integraciones corporativas:** Los departamentos legales corporativos necesitan integrar datos judiciales en sus sistemas internos (ERP, CRM) con autenticación estándar y tokens OAuth2. Ningún competidor lo ofrece documentadamente.

## 5. La ventaja real

**API con autenticación enterprise al PJUD — la única oferta estructuralmente diferenciada en el mercado**

lau-ai es el único actor que combina: (1) acceso programático documentado a datos del PJUD chileno, (2) autenticación Auth0/OAuth2 estándar para integraciones enterprise, y (3) cobertura de todas las jurisdicciones en tiempo real. Boostr.cl tiene API pero sin SLA, sin Auth0, sin soporte enterprise, y en modelo pay-per-use sin garantías. CaseTracking tiene conexión directa al PJUD pero no la expone como API consumible — es una caja negra para sus clientes.

Esta ventaja es estructuralmente difícil de copiar porque requiere mantener integración técnica funcional con todos los sistemas del PJUD (que no tienen API oficial pública), cumplir con regulaciones de seguridad de datos judiciales, y construir infraestructura Auth0 con SLA. CaseTracking podría intentar replicarla en 12–18 meses si decide pivotar hacia modelo de plataforma/API, pero su modelo de negocio actual es SaaS de interfaz, no plataforma de datos — el incentivo a canibalizarse es bajo.

## 6. Riesgo competitivo principal

**CaseTracking (LemonTech)**

Es el riesgo más real por tres razones convergentes: (1) ya tiene la integración técnica con el PJUD funcionando a escala (700,000 causas diarias), lo que significa que la parte más difícil de construir ya existe internamente; (2) tiene recursos (800+ clientes, respaldo de LemonTech como empresa madura con ISO 27001); (3) si decide lanzar una API para desarrolladores o un producto white-label, elimina el gap principal de lau-ai en semanas. El riesgo se vuelve real si LemonTech observa tracción de lau-ai en el segmento de integraciones corporativas o si un cliente grande les pide acceso API directamente — ese pedido puede disparar una hoja de ruta que hoy no tienen pero que pueden ejecutar.
